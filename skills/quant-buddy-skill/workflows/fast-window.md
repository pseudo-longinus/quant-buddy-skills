# 快速执行 · 最近 N 日短窗序列/窗口统计

> **适用范围**：单资产，查询最近 N 个交易日的价格/成交序列，或该窗口内的统计量。  
> 进入本文件前必须已满足 SKILL.md 快速路径触发条件。  
> 完整的序列执行规则（受控降级、Backward Recovery 等）见 `workflows/quick-window.md`，本文件为执行主干。

---

## 执行步骤

```
① newSession
→ ② confirmMultipleAssets
→ ③ runMultiFormula（生成一维时间序列）
→ ④-A [仅需统计量] 从 description 末值直接提取 → ⑤
→ ④-B [需完整序列] readData(mode="last_column_full", start_date=<当前日期-3×N天>) → 取末尾 N 行 → ⑤
→ ⑤ 输出最终答案
```

**停止条件**：所需 N 行数据（或统计末值）已到手 → 立刻停止。

---

## 字段公式模板

| 字段 | 公式 |
|------|------|
| 收盘价序列 | `收盘 = 收盘价(资产名)` *(首选)* / `收盘 = "全市场每日收盘价" * 取出(资产名)` *(备选)* |
| 每日涨跌幅序列 | `日涨幅 = 涨跌幅("收盘", 1)` *(首选)* / `日涨幅 = "全市场每日回报率" * 取出(资产名)` *(备选)* |
| 成交量序列 | `成交量 = "全市场每日成交量" * 取出(资产名)` |
| 成交额序列 | `成交额 = "全市场每日成交额" * 取出(资产名)` |
| 最近 N 日窗口最高 | `窗口最高 = 最大("全市场每日最高价" * 取出(资产名), N)` |
| 最近 N 日窗口最低 | `窗口最低 = 最小("全市场每日最低价" * 取出(资产名), N)` |

**runMultiFormula begin_date 规则（硬规则，防止 token 超限）**：

| 用户 N | begin_date 设为（相对今天回溯） |
|--------|-------------------------------|
| N ≤ 20 | 今天 − 4 个月 |
| 21 ≤ N ≤ 60 | 今天 − 8 个月 |
| N > 60 | 安全失败，告知用户窗口过大 |

> ⚠️ 禁止使用 `20240101` 等超过 1 年前的 begin_date——会导致 token 超限。

**runMultiFormula 其他固定参数**：
```json
{"include_description": true, "use_minute_data": true}
```

## readData 规则（路径 B 必须遵守）

- 模式：`last_column_full`（一维序列唯一可用模式）
- 传 `start_date` = 当前日期 − 3×N 个自然日（防止拉取过量数据）
- 从返回时间序列末尾取 N 条，忽略其余
- ⛔ 禁止 `smart_sample` / `last_n_rows` / `table_data`（一维序列不支持）

---

## 取值与统计规则

- **只需统计量**（最高/最低/振幅/区间收益）→ 优先从 description 末值提取，无需 readData
- **区间收益** = `(末值 − 首值) / 首值 × 100%`（基于窗口首行和末行原始值计算）
- **振幅** = `(窗口最高 − 窗口最低) / 窗口最低 × 100%`
- 涨跌幅小数 → ×100 加 `%`
- 日期/年份必须与工具返回原始值逐字一致，不得凭记忆填写

---

## 保护规则（4 条）

1. **evidence-only**：只输出工具返回的数值；禁止推断归因或引用窗口外数据
2. **去过程化**：首句必须是资产名 + 数据结论；禁止「已成功获取」「数据已获取」等任何变体
3. **仅窗口内计算**：所有统计只能基于读取到的 N 行，禁止引用窗口外数据点
4. **序列展示**：按日期升序输出，禁止中间暴露"等等，需要重新排序"等草稿修正话术
